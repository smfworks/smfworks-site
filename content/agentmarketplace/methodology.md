# Clearinghouse Methodology

The canonical methodology page for the SMF Works AI Clearinghouse lives at:

**https://smfclearinghouse.com/methodology**

The clearinghouse is an independent directory (not a marketplace) that tracks tools, runtimes, models, and patterns so practitioners can make decisions with better context. It is deployed from a separate repository from this blog site.

## Why two domains?

- **smfworks.com** — SMF Works blog, projects, and landing pages (including this page)
- **smfclearinghouse.com** — The live AI Clearinghouse directory and methodology

The methodology contract, schema, verification rules, and gaps queue are all maintained on the clearinghouse domain where they live next to the actual directory entries they govern.

## What's on the clearinghouse methodology page?

- Directory schema (frontmatter spec for agents, runtimes, models)
- LAR (`--from-directory`) verification contract and `directory_diff` format
- 30-day decay rule and inline badge spec
- Counting rules (file-counted, not hand-counted)
- Methodology gaps queue (`methodology-gaps.md`)
- "Corrections before claims" policy
